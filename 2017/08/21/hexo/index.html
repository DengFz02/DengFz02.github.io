使用 Hexo 搭建博客并部署于 Github

前言

我常年混迹与酷安，在里面浏览了许多大牛的博客，里面精美的页面以及深奥的文章让我深深震撼，同时我也深谙网页的奇妙与美丽。于是兴趣被点燃，开始了自己的搭建博客之路。

什么是博客？

博客，是 Blog 的音译，中文译名为“网络日志”。具体参见百度百科：什么是博客？

这里有些文章以供参考：为什么你要写博客？ 如何搭建独立博客？

准备工具

  • Git
  • Node.js
  1. 进入git官网，下载对应系统版本的的git。

    下载完成后直接点击 Next 进行安装。
  2. 进入Node.js官网。这里有两个版本供我们选择：

    左边的版本可以理解为稳定版，推荐给大多数用户，右边的版本为开发板，具有最新的功能。我们建议下载左边的版本。
    下载完成后，点击 Next 进行安装。

    安装完成后，至此，所有的准备工作完成。

安装 Hexo

建议在官网中阅读中文文档进行安装。

  1. 右击鼠标，点击 Git Bash Here.
  2. 键入 $ npm install hexo-cli -g即可进行安装。
  3. 创建你的 Blog 文件并打开，右击鼠标打开Git，键入 $npm init即可进行初始化。接着输入 $npm install即可完成安装。

Hexo部分已经完成，接下来是GitHub部分。

GitHub 部分

  1. 进入GitHub官网，创建你的 GitHub 账号。
  2. 新建一个respository.
  3. 输入你的 respository name, 注意！这里需输入 name + github.io, name与你的 Owner 名字相同。我参考了网络上的教程，有些说直接输入name就行，这样也可以创建博客，但是需要在 Settings 中手动开启 GitHub Pages 并且打开网页后是全文字网页。（没有图片）十分难看，明显是出错了。

    （图中提示我已经注册。）
  4. 复制你的 respository 地址，至此 GitHub 部分暂告一个段落。

回到 Hexo

  1. 打开你的 Blog 文件，点击_config.yml文件进行设定。
  2. 找到最后一行的deploy，并照图中的进行编辑。

    其中，repo地址为你新创建的 respository 地址，粘贴就好。至此，基础设置完成。

部署到 GitHub

在你的 Blog 文件中打开 Git, 输入 $ hexo g生成静态文件，接着，输入 $ hexo s即可生成本地预览。

其中，http://localhost:4000/为你的本地博客地址，在浏览器中打开即可看到你的博客全貌。最后，输入 $ hexo d即可部署到你的 GitHub 上。打开 GitHub，进入Settings可查看你的域名。

蓝色链接即为你的域名。本教程结束。
